### Key design decisions

- **Error propagation**: `die` inside `parse_manifest` is called via `$()` command substitution (not `< <()`), so errors propagate. The `|| die` pattern handles the `set -e` suppression that occurs inside functions called as part of `||` chains.
- **Regex fix**: Stored the requirements.txt operator regex in a variable to avoid bash parsing `<`/`>` as operators inside `[[ =~ ]]`.
- **Exit codes**: 0 = all approved, 1 = error, 2 = denied licenses found.
- **Act**: Uses `--pull=false` to use the locally available `act-ubuntu-pwsh:latest` image.
