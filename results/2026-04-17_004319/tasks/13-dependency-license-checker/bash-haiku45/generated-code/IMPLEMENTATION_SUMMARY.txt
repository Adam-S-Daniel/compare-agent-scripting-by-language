================================================================================
USAGE EXAMPLES
================================================================================

Parse package.json without config:
  ./dependency-license-checker.sh --manifest package.json

Parse with license configuration:
  ./dependency-license-checker.sh --manifest package.json --config license-config.json

Parse requirements.txt:
  ./dependency-license-checker.sh --manifest requirements.txt --config license-config.json

Configuration file (license-config.json):
  {
    "allowlist": ["MIT", "Apache-2.0", "BSD-3-Clause"],
    "denylist": ["GPL-3.0", "AGPL-3.0"]
  }

Output example:
  DEPENDENCY                VERSION         LICENSE         STATUS
  ===============================================================================
  lodash                    4.17.21         MIT             APPROVED
  express                   4.18.2          MIT             APPROVED
  mystery-lib               1.0.0           Unlicense       UNKNOWN
  viral-lib                 1.0.0           GPL-3.0         DENIED

================================================================================
IMPLEMENTATION DETAILS
================================================================================

TDD Methodology:
- Each feature was driven by a failing test
- Minimum code was written to make tests pass
- Mock license database enables testing without external dependencies
- Tests are isolated and repeatable

Key Features:
- Dual format support (package.json and requirements.txt)
- Configurable allow/deny lists in JSON format
- Three-level status reporting (APPROVED/DENIED/UNKNOWN)
- Graceful error handling with meaningful messages
- Mock license database for testing (easily extensible)

Design Decisions:
- Used jq for JSON parsing (robust and standard)
- Associative array for mock license database
- Table format output for readability
- Exit code 0 on success, 1 on error
- Shellcheck compatible code
