2. **`parse_commits`** — parses `git log --oneline`-style text into `[{hash, message}]`; handles multi-line commit bodies (for `BREAKING CHANGE` footers).
3. **`determine_bump_type`** — scans commits, returns `"major"` / `"minor"` / `"patch"` using priority: `BREAKING CHANGE` or `type!:` > `feat:` > everything else.
4. **`bump_version`** — pure semver arithmetic; resets lower components on minor/major bumps.
5. **`update_version_file`** — writes new version back, preserving all other JSON fields.
6. **`generate_changelog_entry`** — groups commits into `### Breaking Changes / Features / Bug Fixes / Other Changes` sections.
