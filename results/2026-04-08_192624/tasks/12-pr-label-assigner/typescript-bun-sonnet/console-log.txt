### Key design decisions

- **Glob matching**: Custom implementation supporting `**` (any depth), `*` (within segment), `?` (single char), `.` (literal). The `**` handling uses a placeholder approach to avoid double-escaping conflicts.
- **Priority ordering**: Rules sorted by priority descending; higher-priority rules appear first in `matchedRules` output.
- **Exclusive flag**: When a rule has `exclusive: true`, it suppresses lower-priority rules for the same file (e.g., `src/api/**` with exclusive blocks `src/**`).
- **Multiple labels per file**: All matching rules fire unless blocked by an exclusive rule.
- **Act tests assert exact values**: Each test case checks the exact set of labels in `LABELS_OUTPUT=...`, not just that output appeared.
