### Key TDD bug caught in refactor

The initial git log format used `|||` as a record separator. When `%b` (body) is empty, git concatenates entries without spacing, turning `|||` into `||||` which bled the pipe into the next hash field — causing `feat:` commits to be mis-parsed as `patch`. Fixed in the refactor step by switching to NUL (`%x00`) separators, which cannot appear in commit messages.
