## Implementation structure (TDD order)

**Section 1 — Queue** (`New-Queue`, `Get-NextQueueItem`)  
Uses `List<object>` for a mutable FIFO queue. Tests verify creation and FIFO dequeue order.

**Section 2 — Dead-letter queue** (`New-DeadLetterQueue`, `Add-ToDeadLetterQueue`)  
Stores `{Item, Reason}` entries. Tests verify items + reasons are recorded correctly.

**Section 3 — Mockable processor** (`Invoke-ProcessItem`)  
Thin wrapper around a user-supplied `[scriptblock]` — catches all exceptions and returns `$true`/`$false`. Fully mockable via scriptblock injection.

**Section 4 — Exponential backoff retry** (`Invoke-WithRetry`)  
Loop from `retry=0` to `MaxRetries` (inclusive = 1 initial + N retries). Delay formula: `BaseDelayMs × 2^retry`. Returns `{Success, Attempts, Delays[], Error}`. Tests verified delay values `[100, 200, 400]` for `BaseDelayMs=100, MaxRetries=3`.

**Section 5 — Progress reporting** (`New-PipelineProgress`, `Update-PipelineProgress`, `Write-PipelineProgress`)  
Mutable `PSCustomObject` with `Total/Processed/Failed/Retrying` counters. `RetryStart`/`RetryEnd` events bracket each processing attempt.

**Section 6 — Pipeline orchestration** (`Invoke-Pipeline`)  
Drains the queue, runs each item through `Invoke-WithRetry`, routes successes to `Processed` and permanent failures to the DLQ. Returns a structured summary with a human-readable `SummaryText`.

**Key design note:** Tests use hashtables (`@{ attempts = 0 }`) for shared mutable state in scriptblock mocks — plain integer variables would shadow rather than mutate across scriptblock invocation boundaries in PowerShell.
