### TDD Cycles followed

| Cycle | Class/Feature | Tests |
|-------|---------------|-------|
| 1 | `MockQueue` — injectable FIFO queue | dequeue, empty→None, enqueue, size |
| 2 | `RetryPolicy` — exponential backoff | doubles each attempt, max_retries cap, max_delay cap |
| 3 | `ProgressReporter` — real-time counters | initial zeros, processed/failed/retrying counters, summary, callback |
| 4 | `Pipeline` — full orchestration | success, DLQ, transient success, sleep delays, DLQ error message, mixed, multi-fail |
| 5 | `PipelineResult` — value object | duration field, summary with duration, DLQ listing in summary |

### Key design decisions

- **All queue/processor operations are injected** — `MockQueue`, `processor fn`, `RetryPolicy`, and `ProgressReporter` are all passed into `Pipeline`, making every piece independently mockable.
- **`time.sleep` patched via `@patch("pipeline.time.sleep")`** — tests run instantly.
- **`RetryPolicy.jitter=False`** in tests — deterministic backoff assertions; production use enables jitter to avoid thundering herd.
- **DLQ entries** carry `{"item": …, "error": str(last_error)}` for observability.
