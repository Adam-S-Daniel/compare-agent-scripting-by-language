## Module Functions (TDD Cycles)

1. **`Get-ProcessData`** — Retrieves process info via an injectable `-DataSource` scriptblock. Defaults to live `Get-Process` data; tests inject mock data for determinism.

2. **`Filter-ProcessByThreshold`** — Filters by `-CPUThreshold` and/or `-MemoryThresholdMB`. When both are 0 (default), returns all. When one is set, filters by that criterion. When both set, uses OR (union).

3. **`Get-TopConsumers`** — Returns top N processes sorted by CPU (default) or Memory descending. Validates TopN > 0 and SortBy in (CPU, Memory).

4. **`New-AlertReport`** — Generates a formatted alert report with header, config, timestamp, process table, and alert count. Combines filtering + top-N internally.

## Key Design Decisions

- **Mockability**: The `-DataSource` scriptblock pattern lets tests inject deterministic data without touching live system state
- **Array preservation**: Unary comma operator (`, $array`) prevents PowerShell from unwrapping empty/single-element arrays on return
- **Linux compatibility**: Default data source wraps each process in try/catch since `StartTime`/`CPU` may be unavailable
- **`:F1` format**: Avoids locale-dependent thousands separators that would break regex assertions
