### File Structure

```
ProcessMonitorLib/               ← library (all business logic, testable)
  Models.cs                      ← ProcessInfo, ThresholdConfig, AlertReport, etc.
  IProcessProvider.cs            ← interface (inject mocks in tests, SystemProcessProvider in prod)
  Services/ProcessMonitorService.cs  ← FilterByCpu, FilterByMemory, GetTopN, GenerateReport
  SystemProcessProvider.cs       ← live OS implementation (never used in tests)

ProcessMonitor.Tests/            ← xUnit test project
  ProcessMonitorTests.cs         ← 18 tests across 6 test classes

ProcessMonitor.cs                ← .NET 10 file-based app entry point
  #:project ProcessMonitorLib/   ← references library via .NET 10 directive
```

### Key Design Decisions
- **`IProcessProvider`** decouples live OS reads from business logic — tests inject `MockProcessProvider` with controlled data, zero live system dependency
- **Thresholds use strict `>` comparison** — a process at exactly the threshold is not alerted
- **`ValidateConfig`** catches bad configs before any processing begins
- **`Environment.Exit(1)`** on alerts — useful for shell scripting pipelines
