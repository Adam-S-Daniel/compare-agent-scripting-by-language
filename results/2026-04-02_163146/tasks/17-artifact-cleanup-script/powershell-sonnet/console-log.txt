**TDD Cycles:**

| Cycle | Test Describe block | What it tests |
|-------|---------------------|---------------|
| 1 | `Get-MockArtifacts` | Mock fixture returns valid artifacts with all required fields |
| 2 | `Invoke-MaxAgePolicy` | Marks artifacts older than MaxAgeDays; exact boundary is retained |
| 3 | `Invoke-MaxSizePolicy` | Deletes oldest artifacts until total size ≤ limit; skips pre-marked |
| 4 | `Invoke-KeepLatestNPolicy` | Keeps N newest per workflow, deletes excess |
| 5 | `Invoke-RetentionPolicies` | Chains all three policies in sequence |
| 6 | `New-DeletionPlan` | Produces structured plan with `SpaceReclaimedBytes` and `Summary` |
| 7 | `Invoke-ArtifactCleanup` | Top-level entry: dry-run flag, error on bad policy |

**Key design decisions:**
- `ReferenceTime` parameter on age-based functions makes tests deterministic (no wall-clock drift)
- `ConvertTo-ArtifactObject` normalizes both hashtables and PSCustomObjects, initializing `MarkedForDeletion = $false` and `DeletionReasons = @()`
- All three policies use reference-equality on the same PSCustomObjects (mutations propagate through the pipeline without copying)
- `Confirm-Policy` validates required keys upfront with a descriptive error message
- Dry-run is fully supported — plan is generated but no real deletions occur (a production implementation would call the GitHub API here)
