### Key design decisions

- **Effective-size validation**: uses Cartesian product minus excluded combinations — so adding excludes can rescue a matrix that exceeds the limit
- **JSON output**: produces `{ "strategy": { "matrix": {...}, "fail-fast": bool, "max-parallel": N } }` exactly matching GitHub Actions format; `max-parallel` is omitted when null
- **Exclude matching**: a job is excluded only when ALL key-value pairs in an exclude rule match (same semantics as GitHub Actions)
