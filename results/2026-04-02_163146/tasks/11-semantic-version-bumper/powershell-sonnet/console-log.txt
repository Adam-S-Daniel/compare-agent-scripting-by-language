**Functionality:**

| Function | Purpose |
|---|---|
| `Get-CurrentVersion` | Reads semver from plain text file or `package.json` |
| `Get-BumpType` | Analyses conventional commits → `major`/`minor`/`patch`/`none` |
| `Get-NextVersion` | Computes next semver from current + bump type |
| `Set-Version` | Writes updated version back to file (plain text or JSON) |
| `New-ChangelogEntry` | Generates Markdown changelog grouped by type |
| `Update-Changelog` | Prepends entry to `CHANGELOG.md` (creates if absent) |
| `Invoke-VersionBump` | Orchestrator: read → bump → write → changelog |

**Mock commit fixtures** defined in the test file cover: patch-only, minor, major (`feat!:` prefix), major (BREAKING CHANGE body), and no-release commits.

**Key design notes:**
- JSON detection uses `StartsWith('{')` rather than try/catch on `ConvertFrom-Json`, since that cmdlet throws a `RuntimeException` subclass that would conflict with intentional error re-throws
- `Get-BumpType` returns immediately with `"major"` on first breaking change seen for efficiency
- All file I/O is isolated in tests via `New-TemporaryFile`/temp directories for true unit test isolation
