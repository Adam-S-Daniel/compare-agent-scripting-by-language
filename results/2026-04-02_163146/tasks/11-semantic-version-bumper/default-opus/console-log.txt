## TDD Cycles

1. **parse_version / format_version** — Parse `"v1.2.3"` → `(1, 2, 3)` and back
2. **classify_commit** — Map conventional commit prefixes to bump types (`fix:` → PATCH, `feat:` → MINOR, `!` or `BREAKING CHANGE` → MAJOR)
3. **determine_bump** — Pick highest bump from a list of commits
4. **bump_version** — Apply MAJOR/MINOR/PATCH to a version tuple
5. **read/write version files** — Support both plain `VERSION` files and `package.json`
6. **generate_changelog** — Markdown changelog grouped by type (Breaking, Features, Bug Fixes, Other)
7. **run_pipeline** — End-to-end integration: read → classify → bump → write → changelog
