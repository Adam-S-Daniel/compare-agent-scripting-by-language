### TDD cycles

**Cycle 1 — `Read-DependencyManifest`**: Parses `package.json` (runtime + optional dev deps) and `requirements.txt` (pinned versions, skips comments). Throws on missing files (`*not found*`) and unsupported formats (`*unsupported*`).

**Cycle 2 — `Test-LicenseCompliance`**: Returns `approved` / `denied` / `unknown`. Deny-list takes priority. Case-insensitive. Empty license → `unknown`.

**Cycle 3 — `Invoke-LicenseLookup`**: Built-in mock database covers common packages. Returns empty string for unknowns. Injected via scriptblock parameter in higher-level functions.

**Cycle 4 — `New-ComplianceReport`**: Orchestrates parsing + lookup + classification. Returns a hashtable with `Dependencies[]` (Name, Version, License, Status) and `Summary` (Total, Approved, Denied, Unknown, IsCompliant).

**Cycle 5 — `Export-ComplianceReport`**: Writes `text` (formatted table) or `json` output to a file.

- `Set-StrictMode -Latest` + `$ErrorActionPreference = 'Stop'` in every file
- All parameters are explicitly typed (`[string]`, `[hashtable]`, `[scriptblock]`, `[switch]`)
- All functions have `[CmdletBinding()]` and `[OutputType()]`
- Explicit casts throughout (`[string]`, `[int]`, `[PSCustomObject[]]`)
