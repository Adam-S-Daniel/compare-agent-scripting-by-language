| 4 | Verification queries | 8 tests: orphan checks, revenue, price accuracy, `verify_all()` structure |

**Key design decisions:**
- `random.Random(seed)` instance isolated from global state → deterministic, reproducible data
- Each test gets a fresh in-memory SQLite DB (`:memory:`) → full isolation, no test ordering issues
- `generate_users` deduplicates emails with a counter suffix — handles collision in small name spaces
- `generate_orders` fetches product prices from DB before the loop — total_price matches exactly
- `verify_all()` returns a structured dict so tests can assert individual check names
