## TDD cycle followed

Each piece of functionality was driven test-first:

1. **Version parsing** — `parse_version()` handles `version.txt` and `package.json`, raises on bad input
2. **Commit classification** — `determine_bump_type()` maps `feat!`/`BREAKING CHANGE` → major, `feat` → minor, `fix` → patch
3. **Version calculation** — `bump_version()` applies semver rules (minor resets patch, major resets both)
4. **File update** — `update_version_file()` writes back, preserving all other `package.json` fields
5. **Changelog generation** — `generate_changelog_entry()` produces dated markdown with sections (Breaking Changes / Features / Bug Fixes)
6. **Integration** — `run_version_bumper()` runs the full pipeline end-to-end
7. **Workflow structure** — 8 tests verify triggers, checkout, Python setup, script references, and `actionlint` lint
