### Key design decisions

- **`ProcessInfo`** is a `@dataclass` — cheap to construct, easy to snapshot in fixtures
- **`read_processes(provider=)`** — injectable provider callable means tests never touch the real OS; the default `_live_provider()` uses `psutil` (lazy-imported so the library loads without it)
- **`filter_by_threshold`** — OR logic (flag if CPU *or* memory exceeds threshold); validates inputs are in [0, 100]
- **`top_n_consumers`** — ranks by `cpu% + memory%` combined score, descending
- **`generate_alert_report`** — returns a `dict` with `alert_count`, `alerts` (structured list), and `formatted` (human-readable string); each alert entry includes a `reasons` list explaining which threshold was breached
