**Strict mode compliance in `LogAnalyzer.psm1`:**
- `Set-StrictMode -Version Latest` + `$ErrorActionPreference = 'Stop'` at top
- Every function has `[CmdletBinding()]`, `[OutputType()]`, and explicitly typed `[string]`/`[datetime]`/`[PSCustomObject[]]` parameters
- All casts are explicit (e.g., `[int]$g.Count`, `[string]$obj.level`)

**Key design note:** Pester 5 intercepts `Set-StrictMode` during its *discovery* phase. The fix was: (1) move `Import-Module` into a top-level `BeforeAll` block so it runs during the *execution* phase, and (2) use `Set-StrictMode -Version Latest` (explicit parameter name) instead of the positional shorthand `-Latest`.
