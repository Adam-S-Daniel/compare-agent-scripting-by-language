**Key design decisions:**
- Policies compose in order: age → keep-latest-N → size cap. Each stage can only move artifacts from retain→delete.
- `run_cleanup()` takes an injectable `delete_fn` callback, making it fully testable without real API calls.
- Dry-run is the default — `--execute` must be explicitly passed to perform deletions.
- Delete failures are caught and reported in the summary (not swallowed, not crashing).
- CLI uses argparse with `--max-age-days`, `--keep-latest-n`, `--max-total-size-mb`, `--dry-run`/`--execute`.
